1. 지역 변수
    1) 블록({}) 내에서 선언된 변수 
        - 블록 내에서만 사용 가능 
        - 블록이 종료되면 메모리 상에서 소멸됨 
        - 매개변수도 지역변수임
        - 초기화하지 않으면 쓰레기 값이 할당됨

2. 전역 변수 
    1) 함수 외부에서 선언된 변수 
        - 모든 함수에서 사용 가능 
    2) 프로그램이 실행될 때 메모리가 할당되며
       프로그램이 종료되면 메모리 상에서 소멸됨 
    3) 초기화하지 않으면 0으로 자동 초기화됨 

3. 정적 변수 
    1) static으로 선언된 변수 
    2) 선언된 블록({}) 내에서만 사용할 수 있음 (지역변수 특성)
    3) 프로그램이 실행될 때 메모리가 할당되며  
       프로그램이 종료되면 메모리 상에서 소멸됨 (전역변수 특성)
       - 함수 종료되더라도 변수 값 유지해야 하는 경우 정적 변수 사용함 

4. 메모리 관리 
1) Memory 
   - 프로그램이 실행되는 동안 데이터를 저장하기 위한 메모리 공간은 구분되어 저장 됨

2) RAM 
   - Stack
      - 함수나 블록내에서 선언된 지역변수, 매개변수 
      - 시스템에 의해 할당/해제
      - 컴파일 타임에 크기 결정
   - Heap
      - 동적으로 할당된 메모리 
      - 프로그래머에 의해 할당/해제
      - 런타임에 크기 결정
      - malloc()/new, free()/delete()
   - Data
      - 전역변수나 정적변수(static 변수)
      - 프로그램 실행/종료까지 메모리 할당 
      - String Pool (문자열 리터럴 상수)
      - Literal Constants (정수, 실수, 문자, 리터럴 상수)
      - Symbolic Constants

3) ROM 
   - Code/Text 영역  
      - Functions (함수의 명령어 저장, 프로그램의 실행 코드)
      - Symbolic Constants
         - 읽기 전용 상수
